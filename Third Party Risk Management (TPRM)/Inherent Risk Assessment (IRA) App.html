How this app works

  • Assess — A Service Owner records the vendor details and answers the IRQ. As answers are selected, the app shows the inherent-risk rating and the exact DDQ domains/focus areas that are scoped in, with the reason each was triggered.
  • Conditional questions appear only when their display condition is met (e.g. CYB.02 only shows when CYB.01 is not "Not Applicable").
  • Records — Saved assessments live in this browser (localStorage). Reopen, duplicate, export, or delete them. Use Export all to back up.
  • Questionnaire Builder — Edit every question, option, trigger, rule, domain and rating band without touching code. This is how you keep the app current.

Updating after the IRQ changes

  • Update from Excel (.xlsx) — Re-import an updated IRQ spreadsheet. The app matches rows by Question ID and refreshes the question text, options, response type and guidance, while preserving the trigger rules and scores you already configured for matching IDs. New IDs are added; missing ones are flagged for review.
  • In-app editing — Add/edit/remove questions, options, per-option triggers, cross-question rules, domains and bands directly.
  • Import / Export JSON — The whole questionnaire is one JSON config. Export it to keep versioned copies; import to roll forward or share across the team.
Excel re-import refreshes question content but cannot infer structured trigger logic from free text. After importing, review per-option triggers for any new/changed options in the Builder.

Excel format expected

First data columns, in order: Q. No., Risk Area, Question, Configuration, Response Type, Response Options, User Guidance, Trigger Logic, Scoring. Options within a cell are separated by new lines (e.g. 1) Yes / 2) No).
